### Outline

-There are mockups in the docs/mockups dir that contain the 3 different pages for this SPA

- The ui should use material design

- It should allow both dark or light mode, defaulting to whatever the user's OS is set to

- The color scheme should lean toward neutral and cool colors. When using warmer colors, don't use bright colors. aim for a more muted palette.

- The website should be accessable but will have a limited number of potential users. Follow WCAG standards at Level AA.

- The web ui should render correctly on both mobile, PC and tablets. Prioritize PC when it comes to design, but mobile should still be at least a usable experience

- Maybe use tailwindcss to make consistent styling easier?
